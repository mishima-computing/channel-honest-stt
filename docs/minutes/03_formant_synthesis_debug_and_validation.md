# 母音合成器（フォルマント合成）のデバッグと定量検証

## 1. 背景と問題の所在
「02_degradation_verification_report」におけるスペクトログラム目視検証にて、以下の重大な異常が指摘された。
1. **`/i/` Cleanの異常**: 300Hzと2300Hzにエネルギーが集中せず、全帯域が一様に明るく倍音が並んでいるだけに見える（フォルマントの山と谷が形成されていない）。
2. **`/u/` と `/o/` の異常**: F2（1300Hz, 900Hz）でも想定F3でもない2500Hz付近に説明不能な強いエネルギー帯が存在する。

これらの異常から、劣化による「にじみ」を評価する以前に「そもそも合成器が正しく母音を作れていない（Cleanが壊れている）」という前提の問題が浮上した。

## 2. 原因と修正（カスケード接続からパラレル接続へ）
原因は、`scipy.signal.iirpeak` を用いた共振フィルタを**直列（カスケード）接続**していた点にある。
`iirpeak` は、指定したピーク以外の全帯域のゲインをゼロ近くまで強烈に落とす特性を持つ。これを直列に繋ぐと、後段のフィルタが前段のフィルタのピークを殺してしまい、結果的にすべてのフォルマントが潰れてフラットな特性になる、あるいは最も高域のピークのみが変な形で残るという現象を引き起こしていた。

**【修正内容】**
フィルタの接続方式を**並列（パラレル）接続**に変更した。
各フォルマントフィルタ（F1, F2, F3）を独立してインパルス列ソースに適用し、それぞれの出力を振幅比（F1 > F2 > F3）をつけて足し合わせるアーキテクチャ（Parallel Formant Synthesizer）へと書き換えた。これにより、各フォルマントが互いに干渉することなく独立して共振ピークを形成可能になった。

## 3. 定量評価（真の伝達関数による数値検証）
合成器の健全性を目視だけでなく「数値」で定量的に立証するため、インパルス列の倍音干渉を排除した**「合成器の真の周波数特性（伝達関数）」**から数学的にピークを抽出するテスト（`02_formant_numeric_check.py`）を実施した。

| Vowel | Target F1 | Measured F1 | Target F2 | Measured F2 |
|-------|-----------|-------------|-----------|-------------|
| /i/   | 300       | 299         | 2300      | 2300        |
| /e/   | 450       | 451         | 2000      | 2001        |
| /a/   | 700       | 697         | 1200      | 1207        |
| /o/   | 500       | 498         | 900       | 908         |
| /u/   | 350       | 349         | 1300      | 1304        |

結果として、全5母音において実測ピークがターゲット値と**誤差数Hz以内で完全に一致**した。これにより「合成器は設計通りのフォルマントを持つ母音を正確に生成できている」という前提が数字で立証された。

## 4. 再検証結果と結論
合成器の正常性が立証されたため、再度スペクトログラムをdBスケール（対数表示）にて出力し直した。

* **`/i/` のF1減衰**: Clean状態における300Hzと2300Hzの明確な山のコントラストが確認でき、Degraded状態では500HzハイパスによりF1（300Hz）が明確に減衰している様子が観測された。
* **`/u/` vs `/o/` の境界**: 合成が正常になった上で改めてDegraded状態を見ると、μ-lawの非線形量子化ノイズとtanhのクリッピングにより、F2（1300Hz vs 900Hz）のピーク周辺に激しいにじみが発生し、両者の弁別マージンが著しく狭まっている（接近している）ことが視覚的にハッキリと確認できた。

**【結論】**
デバッグと数値検証により合成器の健全性が担保された状態での「にじみ・混ざり」であるため、**「u-o境界の井戸は畳む（統合する）べきである」というPhase 1の設計判断が物理的にも正しいことが最終確認された**。
